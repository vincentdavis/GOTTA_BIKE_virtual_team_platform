Method & calculations

Each rider is reduced to their power-duration curve (5s, 15s, 30s, 1m, 2m, 5m, 20m best watts from Zwift Racing, 90-day) plus weight and height. For a climb of a given length and grade we solve how long that rider would take, then compare riders.

1. Power needed to climb at a speed

The standard cycling power model — the rider power to hold ground speed v (m/s) on gradient g with total mass m (rider + bike) is:

P = ( Crr·m·G·v  +  m·G·g·v  +  ½·ρ·CdA·v³·draft ) / drivetrain

The three terms are rolling resistance, gravity (dominant on a climb), and aerodynamic drag. G = 9.807 m/s². On steep climbs the aero term is small, so drafting helps little.

2. CdA (drag area)

Estimated from height and weight, CdA = coef · height_m^0.725 · weight_kg^0.425 (a DuBois-style frontal-area approximation). The coefficient and exponents are configurable in Site Config; road/climbing racing uses STD_CDA_COEF (a more upright position than the TTT aero tuck), and each matchup/plan can override the coefficient.

3. Climb time (fixed-point solve)

A climb's duration sets which part of the power curve applies, but that power sets the speed, which sets the duration — so we iterate: guess a time → read sustainable power at that duration (log-time interpolation of the curve) → invert the power model for speed on the grade → recompute the time. This converges in a few steps to each rider's steady best-effort climb time.

4. What the cells show

The metric shown in each cell is described above the table. Because riders' power curves decline at different rates, the ranking changes with climb length — short walls favour punchers, long climbs favour sustained-power riders.

Caveats

  • Power is 90-day bests — fresh and optimistic, not fatigued mid-race.
  • Steady-state effort only; it ignores the surges that often cause the actual drop.
  • CdA is estimated from height/weight, not measured.
  • The grid assumes no draft (each rider riding their own pace).
